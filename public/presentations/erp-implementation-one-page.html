NICSI ERP Implementation - One Page Summary

Execution Snapshot | Date: 28 Feb 2026

Target Module Stack

  • AUTH: Identity, role claims, policy enforcement.
  • MASTER: Canonical reference data (state/UT, department, categories).
  • PI: Project lifecycle, milestones, program KPIs.
  • Vendor Management: Empanelment and compliance flow.
  • Notification: Event-driven alerts and SLA reminders.
  • Accounts: Budget, invoice, payment status governance.
  • Zoho Adapter: Controlled sync + reconciliation logs.

Delivery Model (12 Weeks)

Phase Weeks Outcome
Foundation 1-4 Auth hardening, master APIs, contract freeze.
Core Workflows 5-8 PI + Vendor + Notification end-to-end flow.
Finance & Integration 9-12 Accounts + Zoho adapter + pilot go-live.

Key Risks & Controls

  • API contract drift -> Versioned contracts + contract tests.
  • Integration failure -> Retry queue + reconciliation dashboard.
  • Access policy gaps -> Central RBAC governance checks.
  • Data inconsistency -> Master-data stewardship and periodic audits.

Immediate Actions

  • Approve module boundaries and ownership matrix.
  • Freeze phase-1 API specs for AUTH + MASTER services.
  • Start Sprint-1 with weekly architecture review checkpoint.
  • Track KPIs: error rate, SLA response, audit coverage, release quality.
